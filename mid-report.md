## 2. 프로젝트 내용

### 2.1 서론

본 프로젝트는 사용자가 원하는 도시, 여행일정, 예산, 관심사, 동반자 특성 등을 입력하면 자동으로 여행 일정을 생성해주는 여행 계획 LLM 기반 챗봇 시스템을 개발하는 것을 목표로 한다. 기존의 여행 일정 생성 서비스는 정형화된 추천 또는 단순 검색 기반이라 개인 맞춤형 여행 설계에 한계가 있다. 따라서 본 프로젝트에서는 사용자 입력을 JSON 형태로 받아 Flask API로 전달하고, LLM을 호출해 맞춤형 일정을 생성하는 구조를 채택하였다.

이 시스템은 여행 계획 경험이 부족한 사용자에게 실질적인 도움을 주며, 세부 일정·동선 고려·관심 기반 추천을 자동화한다. 최종적으로 웹 기반에서 설계된 챗봇을 통해 사용자 경험을 극대화하는 것을 목표로 한다.

### 2.2 추진 배경 (자료조사 및 요구 분석)

#### 2.2.1. 개발 배경 및 필요성

현대 여행자들은 짧은 시간 안에 많은 정보를 습득해야 하고, 일정 설계 과정에서 과부하를 겪는 경우가 많다. 특히 해외 여행의 경우 정보량이 많고 동선 고려가 어려워 일정 생성에 많은 시간이 필요하다.
여행 일정 챗봇은 이러한 문제를 해결하며 다음과 같은 필요성을 충족한다:

* 개인 맞춤 일정 자동 생성
* 빠른 일정 비교 및 수정 가능
* 동반자 특성(아이 동반 여부 등)에 맞춘 최적화 일정 제공
* 예산 및 이동 거리 등을 고려한 현실적 추천 제공

#### 2.2.2. 선행기술 및 사례 분석

* **기존 여행 서비스:**  
  TripIt, 마이리얼트립, 클룩 등 다양한 여행 일정 추천 서비스는 존재하나 사용자 입력 기반의 맞춤 일정 생성 기능은 제한적

* **LLM 기반 사례:**  
  GPT API를 활용한 일정 추천 예시는 존재하지만, 대부분 단발성 생성에 그치며 API 기반 플로우 구조가 정교하지 않음

* **본 프로젝트의 차별점:**
  - JSON 기반 입력 구조를 명확히 정의하여 일정 생성 품질 제어
  - Flask를 활용한 독립적 API 서버 구축
  - 사용자 UI를 통한 즉각적 테스트 및 반복 개선이 가능
  - 향후 데이터 수집 및 모델 정교화가 가능하도록 확장성 확보

* **선행기술 한계 및 본 프로젝트 개선점:**
  - 기존 서비스는 개인화 수준이 낮음 → LLM으로 고도화된 개인화 제공
  - 단순 텍스트 입력 기반 → 구조화된 JSON 입력으로 안정적인 일정 생성
  - 일정 설명은 가능하지만 ‘시간 배분, 이동 시간 고려’ 기능이 부족 → 모델 설계로 보완 예정

### 2.3 목표 및 내용

#### 2.3.1. 프로젝트 목표

* 사용자의 여행 조건을 기반으로 **도시/기간/관심사/예산/동선 제약** 등을 반영한 일정 자동생성
* Flask API 기반의 웹 서비스로 구현
* JSON POST 요청을 통해 일정 생성 기능을 테스트 및 배포 가능한 형태로 제작
* 챗봇 UI와 연동하여 지속적인 대화 흐름 제공

#### 2.3.2. 개발 내용

* **범위:** 여행 일정 생성 AI 서버 + 웹 테스트 플랫폼
* **구성 요소:**
  - 사용자 입력(JSON)
  - Flask API 서버
  - GPT 기반 일정 생성 엔진
  - 결과 반환(JSON 일정)

* **데이터셋:**  
  사전 구성된 고정 데이터셋은 없으며, API 호출 기반으로 OpenAI 모델을 사용해 직접 생성

* **시스템 구조 블록 다이어그램:**

```
사용자 → 웹 UI → JSON 입력 → Flask API → GPT 일정 생성 → JSON 출력 → 사용자 출력
```

* **시퀀스 다이어그램:**

```
User → Web → Flask POST /plan → LLM 호출 → 일정 생성 → Flask Response → Web
```

#### 2.3.4. 개발 환경

* **개발환경:**
  - Python 3.10
  - Flask
  - OpenAI API
  - JSON 기반 API 구조

* **학습환경:**  
  - 모델 학습 없음 (API 기반)
